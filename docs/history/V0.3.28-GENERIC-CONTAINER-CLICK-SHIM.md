# v0.3.28 Generic Container Click Shim

## Why this patch exists

The v0.3.27 run showed that the player-inventory patch is now installed and active, but chest/container clicks still did not do anything useful.

The important log split was:

```text
player inventory handleClick entry ...
sent legacy player-inventory transaction ...
sent Java inventory snapshot ...
```

for own-inventory clicks, versus:

```text
inventory serverbound container route current javaContainerId=2 currentType=CONTAINER
```

for chest clicks.

That means player inventory clicks were reaching the patched `InventoryContainer`, but chest clicks were routed to ViaBedrock's ordinary base `Container` implementation. The stock base container handler returns `false`, so ViaBedrock only corrects the Java UI instead of creating a Bedrock inventory action.

## What changed

v0.3.28 adds the base container to the ViaBedrock patch set:

```text
net/raphimc/viabedrock/protocol/storage/InventoryTracker.class
net/raphimc/viabedrock/api/model/container/Container.class
net/raphimc/viabedrock/api/model/container/player/InventoryContainer.class
net/raphimc/viabedrock/api/model/container/player/InventoryContainer$ClickSlot.class
```

The patched base `Container.handleClick(...)` supports a first pass for chest-like containers:

```text
- left-click pickup/place/swap
- right-click split/place-one
- number-key hotbar swap
- shift-click transfer to/from the player inventory
```

It also sends a merged Java container snapshot after handled clicks so the frontend can visibly update both the opened container slots and the player-inventory slots shown inside that container window.

## Expected startup

```text
[java-compat] Verified ViaBedrock inventory patch class entries: 4
[java-compat] ViaBedrock inventory patch active: v0.3.28-generic-container-click-shim
```

## Expected test logs

When clicking inside a chest:

```text
[BedrockRealmBridge] generic container handleClick entry type=CONTAINER ...
[BedrockRealmBridge] sent legacy generic-container transaction reason=container_pickup actions=2
[BedrockRealmBridge] sent Java container snapshot reason=container_pickup ...
```

When clicking your own inventory:

```text
[BedrockRealmBridge] player inventory handleClick entry ...
[BedrockRealmBridge] sent legacy player-inventory transaction reason=...
[BedrockRealmBridge] sent Java inventory snapshot reason=...
```

## Still not final

This patch still uses legacy Bedrock normal `inventory_transaction` packets. Modern Bedrock inventory is fundamentally `item_stack_request` / `item_stack_response` based. If the Java UI moves and then snaps back after the Realm replies with authoritative inventory content, the next patch needs to generate modern `item_stack_request` actions instead of relying on legacy transactions.

Crafting output is also still intentionally incomplete. The short-term target is stable inventory movement and chest slot movement first.
