# v0.3.39 modern 2x2 crafting stack requests

This patch keeps the v0.3.38 rollback for normal inventory movement but makes the player 2x2 crafting path use modern Bedrock `item_stack_request` packets by default.

## What changed

- Added live item-stack-id tracking from clientbound `inventory_content`, `inventory_slot`, and `item_stack_response`.
- Enabled a narrow default rewrite only for transactions touching the player 2x2 `crafting_input` container or the 2x2 crafting output commit.
- Kept plain hotbar/main-inventory movement on the legacy v0.3.36/v0.3.38 path to avoid repeating the v0.3.37 regression.
- Kept full generic legacy transaction rewriting behind `NETHERNET_RELAY_REWRITE_LEGACY_INVENTORY_TO_STACK_REQUESTS=true`.
- Added `NETHERNET_RELAY_REWRITE_CRAFTING_TO_STACK_REQUESTS=false` as a rollback switch for the new crafting-only rewrite.
- Updated smoke coverage for crafting-only rewrite behavior and stack-id tracking.

## Expected logs

Normal inventory movement should not print the rewrite log. Crafting-grid movement should print something like:

```text
[bedrock-relay] Rewriting legacy crafting-grid normal inventory_transaction into 2 modern item_stack_request packet(s): legacy_move_commit_to_item_stack_request:take, legacy_move_commit_to_item_stack_request:place
```

Craft output should print:

```text
[bedrock-relay] Rewriting legacy crafting-grid normal inventory_transaction into 2 modern item_stack_request packet(s): legacy_craft_commit_to_item_stack_request:craft, legacy_craft_commit_to_item_stack_request:place
[bedrock-relay] Java/ViaBedrock -> Realm item_stack_request; forwarding block/item interaction upstream: {"name":"item_stack_request","requestCount":1,"actions":["craft_recipe","results_deprecated","consume","take"]}
```
