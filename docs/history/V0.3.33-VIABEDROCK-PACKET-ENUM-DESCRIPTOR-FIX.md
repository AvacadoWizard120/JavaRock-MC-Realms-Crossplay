# v0.3.33 ViaBedrock packet enum descriptor fix

## Problem

v0.3.32 injected successfully and reported five patched ViaBedrock class entries, but the Java client was kicked during join. The client-side error was:

```text
NoSuchFieldError: Class net.raphimc.viabedrock.protocol.ServerboundBedrockPackets does not have member field 'java.lang.Object MOB_EQUIPMENT'
```

The server-side stack placed the crash at:

```text
net.raphimc.viabedrock.api.model.container.player.InventoryContainer.onSlotChanged
```

That method runs during early inventory/hotbar synchronization, before the user has a chance to test crafting.

## Root cause

The patched `.class` files were compiled against lightweight compatibility stubs. Those stubs declared `ServerboundBedrockPackets` fields as `java.lang.Object`. The real ViaBedrock runtime class declares those fields as the enum type:

```text
Lnet/raphimc/viabedrock/protocol/ServerboundBedrockPackets;
```

The JVM field descriptor is part of field resolution. So even though `MOB_EQUIPMENT` exists in the real class, `MOB_EQUIPMENT:Ljava/lang/Object;` does not.

## Fix

v0.3.33 keeps the v0.3.32 local 2x2 crafting output implementation, but replaces the bundled patched class files with versions compiled against the actual `tools/ViaProxy.jar`.

Patched class entries remain:

```text
net/raphimc/viabedrock/protocol/storage/InventoryTracker.class
net/raphimc/viabedrock/api/model/container/Container.class
net/raphimc/viabedrock/api/model/container/player/InventoryContainer.class
net/raphimc/viabedrock/api/model/container/player/InventoryContainer$ClickSlot.class
net/raphimc/viabedrock/api/model/container/player/InventoryContainer$CraftRecipe.class
```

## New validation

`scripts/viabedrock-inventory-tracker-patch-smoke.cjs` now runs a `javap -verbose` check against the bundled `InventoryContainer.class`. It fails if `MOB_EQUIPMENT` is referenced with the stale `Ljava/lang/Object;` descriptor and requires the real ViaBedrock enum descriptor.

## Expected startup

```text
[java-compat] ViaBedrock inventory patch active: v0.3.33-viabedrock-packet-enum-descriptor-fix
[java-compat] Verified ViaBedrock inventory patch class entries: 5
```
