# v0.3.38 Disable Unsafe Stack Request Rewrite

## Why this patch exists

v0.3.37 tried to convert patched ViaBedrock legacy normal `inventory_transaction` commits into modern Bedrock `item_stack_request` packets. The idea matched the native Bedrock packet baseline, but the generated requests were not yet authoritative enough. Live testing showed basic player-inventory movement regressed.

The latest Java log proves the regression boundary:

```text
player inventory handleClick entry ... input=PICKUP
sent legacy player-inventory transaction reason=pickup_direct_commit
Rewriting legacy normal inventory_transaction into 2 modern item_stack_request packet(s)
Java/ViaBedrock -> Realm item_stack_request ... actions=[take]
Java/ViaBedrock -> Realm item_stack_request ... actions=[place]
```

That means the Java-side click shim was still working, but the relay-side rewrite broke the authoritative commit path.

## Changes

- Disables legacy `inventory_transaction` -> `item_stack_request` rewriting by default.
- Keeps the v0.3.36/v0.3.37 Java inventory cursor and 2x2 local crafting UI behavior.
- Keeps `item_stack_response` normalization, including explicit empty `custom_name` and `filtered_custom_name` strings.
- Keeps the modern rewrite implementation available only as an explicit lab toggle:

```powershell
$env:NETHERNET_RELAY_REWRITE_LEGACY_INVENTORY_TO_STACK_REQUESTS = "true"
```

Do not enable that toggle for normal survival testing yet.

## Expected startup

```text
[java-compat] ViaBedrock inventory patch active: v0.3.38-disable-unsafe-stack-request-rewrite
```

## Expected normal inventory behavior

During ordinary inventory movement, the terminal should **not** print:

```text
Rewriting legacy normal inventory_transaction into 2 modern item_stack_request packet(s)
```

If that line appears, the lab toggle is enabled and the v0.3.37 regression path is active again.

## Next protocol work

The next attempt at modern `item_stack_request` generation needs real authoritative stack ids and slot descriptors from native Bedrock recorder windows. The v0.3.37 generated requests often had unknown/zero source stack ids for live inventory slots, which is not safe enough for the Realm authority.
