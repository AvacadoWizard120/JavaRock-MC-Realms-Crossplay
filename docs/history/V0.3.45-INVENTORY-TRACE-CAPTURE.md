# v0.3.45 focused inventory trace capture

This patch makes the packet census more useful for the current crafting/cursor bug.

## What changed

- Packet Census now writes an optional focused inventory trace:
  - `packet-census/inventory-trace-<run-id>.jsonl`
  - enabled with `PACKET_CENSUS_FOCUS_TRACE=true`
- The trace records action-level details for:
  - `item_stack_request`
  - `item_stack_response`
  - `inventory_transaction`
  - inventory/container content and slot updates
  - `mob_equipment`, `player_hotbar`, `crafting_data`, and `unlocked_recipes`
- `item_stack_request` summaries now include source/destination container, slot, count, stack id, recipe network id, and result item summaries.
- `item_stack_response` summaries now include response status, request id, response containers, and response slots.
- The Java relay latest script enables focused traces and raises the per-kind sample limit.
- The native Bedrock recorder latest script enables focused full traces.
- Added `run-bedrock-inventory-baseline-latest.ps1` for a focused native Bedrock crafting baseline.
- Added:
  - `npm run packet-census:inventory-trace`
  - `node scripts/inventory-trace-summary.cjs`

## Why

The latest Java relay run showed the Realm rejecting generated modern stack requests:

```text
request_id=-101 status=7
request_id=-103 status=49
request_id=-105 status=49
```

That means the Java UI is predicting cursor/crafting changes before the Realm accepts the server-authoritative Bedrock request. The next step is to capture the native Bedrock request/response shape for the same action sequence and compare it with the Java relay trace.

## Focused native test

Run:

```powershell
.\run-bedrock-inventory-baseline-latest.ps1 -RealmName "Example Realm"
```

Then in Minecraft Bedrock Edition, connect to:

```text
127.0.0.1:19133
```

Perform one focused sequence:

```text
1. Open inventory.
2. Put one oak log in the 2x2 grid.
3. Take the oak planks result.
4. Try the right-click/drag style distribution if Bedrock exposes an equivalent action.
5. Close the client and stop the recorder.
```

Summarize the capture:

```powershell
npm run packet-census:inventory-trace -- --limit 80
```
