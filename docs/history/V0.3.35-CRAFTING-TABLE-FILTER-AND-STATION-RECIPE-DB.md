# v0.3.35 crafting-table filter and future station recipe DB

## Why this patch exists

v0.3.34 correctly proved that the Java player 2x2 crafting output can load a live recipe database from the Bedrock `crafting_data` packet, but it exported too much. Bedrock encodes furnace/smelting recipes as `type: shapeless` entries with blocks such as `furnace`, `smoker`, `blast_furnace`, `campfire`, `soul_campfire`, `stonecutter`, or `deprecated`.

Because v0.3.34 only checked `type` and ingredient count, it allowed entries like `minecraft:furnace_log_oak` into the Java 2x2 crafting DB. That is why an oak log could match charcoal instead of planks.

## Fix

The player 2x2 crafting export now only includes recipes whose Bedrock recipe block is exactly:

```text
crafting_table
```

Everything else is excluded from `bridge-crafting-recipes-2x2.json`.

## Preserving future smelting/workstation support

Non-crafting-table station recipes are not discarded. They are written to:

```text
viaproxy-run/bridge-station-recipes-future.json
bridge-station-recipes-future.json
```

That file preserves furnace/smoker/blast-furnace/campfire/stonecutter/cartography/deprecated station recipes for later work. The patched ViaBedrock player inventory does not load that file yet; it is only a future data source.

## Additional shaped recipe fix

The packet-census samples show Bedrock shaped recipe inputs in column-major form. For example, a 1x2 recipe may be encoded as one array containing two cells, and a 2x1 recipe may be encoded as two arrays containing one cell each.

v0.3.35 exports shaped patterns in Java row-major order for the existing Java-side matcher. This keeps recipes such as sticks and torches available in the 2x2 grid instead of accidentally skipping vertical patterns.

## Expected logs

```text
[java-compat] ViaBedrock inventory patch active: v0.3.35-crafting-table-recipe-filter-and-station-db
[bedrock-relay] Exported <N> live Bedrock crafting_table 2x2 recipe(s) for ViaBedrock: ...\bridge-crafting-recipes-2x2.json
[bedrock-relay] Preserved <N> non-crafting-table station recipe(s) for future smelting/workstation support: ...\bridge-station-recipes-future.json
```

With the current packet-census sample, the smoke test sees:

```text
crafting_table 2x2 recipes: 2166
future station recipes: 535
```

## Validation

The smoke test now asserts:

```text
minecraft:furnace_log_oak is excluded from bridge-crafting-recipes-2x2.json
minecraft:furnace_log_oak is preserved in bridge-station-recipes-future.json
minecraft:stick remains included in the 2x2 crafting table DB
log/stem -> planks recipes remain included in the 2x2 crafting table DB
column-major 1x2 shaped recipes are exported correctly
```
