# v0.3.40 right-click crafting distribution

This patch fixes the Java inventory cursor path used by right-click placement and `QUICK_CRAFT` drag placement.

## What changed

- Player inventory right-click placement now commits one item at a time while keeping the carried stack tied to its original source slot.
- `QUICK_CRAFT` add-slot packets now place one item into the hovered slot instead of dumping the entire carried stack.
- Generic container click handling uses the same one-item placement behavior for chests and other opened containers.
- The ViaBedrock inventory patch id was bumped so the next bridge launch rebuilds `ViaProxy.inventory-patched.jar`.

## Expected test

In the Java player inventory, pick up planks, hold right click, and drag across the 2x2 crafting grid. Each hovered grid slot should receive one plank, so the crafting table recipe appears instead of the button recipe.
